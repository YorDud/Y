Что делать пошагово (правильный запуск)

Откройте системный терминал
Windows:
Откройте папку проекта в Проводнике, в адресной строке введите cmd и нажмите Enter
или правый клик в папке → “Открыть в Терминале/PowerShell”
macOS: Откройте Terminal (Программы → Утилиты → Terminal)
Linux: Откройте Terminal
Перейдите в папку проекта
В терминале выполните команду cd до вашей папки с проектом, например:

Bash
cd "C:\Users\ВашеИмя\Desktop\y-underwear"
# или macOS/Linux
cd ~/projects/y-underwear
Проверьте, что Node.js установлен

Bash
node -v
npm -v
Если версии выводятся — всё ок.
Если “command not found” — установите Node.js LTS с nodejs.org, перезапустите терминал и повторите проверку.
Установите зависимости

Bash
npm install
Проверьте .env Убедитесь, что .env находится в корне проекта и заполнен:

Bash
PORT=3000
STRIPE_SECRET_KEY=sk_test_...
STRIPE_PUBLISHABLE_KEY=pk_test_...
ADMIN_USERNAME=admin
ADMIN_PASSWORD=change_this_password
JWT_SECRET=super_secret_jwt_key
ALLOWED_ORIGIN=http://localhost:3000
Примечание: Stripe-ключи нужны для оплаты. Без них сайт откроется, но оплата не будет работать.

Запустите сервер

Bash
node server.js
# или
npm start
В терминале должно появиться:
Y — underwear server running at http://localhost:3000

Откройте сайт в браузере
Введите в адресной строке: http://localhost:3000
Не открывайте index.html напрямую двойным кликом (file://…) — фронтенд должен работать через сервер.
Используйте админку для добавления товаров
Откройте http://localhost:3000/admin.html
Войдите логином/паролем из .env
Загрузите изображение, добавьте товар в коллекцию со статусом now
Проверьте корзину и оплату
Добавьте товар в корзину → Оформить → Stripe Checkout
Для теста: карта 4242 4242 4242 4242, дата в будущем, любой CVC
Если вы всё же видите ReferenceError в браузере

Проверьте, что вы не вставляете команды node/npm в консоль браузера (DevTools).
Убедитесь, что ни один .html не содержит тег <script src=“/server.js”> или чего-то подобного. server.js — это только для Node, он не должен грузиться в браузер.
Откройте DevTools → Console и посмотрите, на какой строке/в каком файле возникает ошибка. Если это просто результат того, что вы ввели “node” или “npm” в консоли — игнорируйте, вводите эти команды только в терминале системы.
Дополнительно

Автоперезапуск при разработке:

Bash
npm i -D nodemon
npx nodemon server.js
Если открываете сайт с другого устройства в локальной сети: используйте http://IP_вашего_ПК:3000 и поменяйте ALLOWED_ORIGIN в .env на этот адрес, перезапустите сервер.































Как это использовать (пошагово)

Подключить ЮKassa и включить Яндекс Pay
Регистрируетесь в ЮKassa (как магазин).
Получаете shopId и secretKey.
Включаете нужные способы оплаты (в т.ч. Яндекс Pay) в кабинете ЮKassa (или через менеджера/модерацию).
Вставляете YOOKASSA_SHOP_ID и YOOKASSA_SECRET_KEY в .env.
Важно: “Яндекс Pay появится” на странице оплаты ЮKassa только если он подключён вашему магазину и доступен для валюты/страны/условий.

Подключить Яндекс.Карты для точки на карте
Создаёте API key в кабинете Яндекса для Maps API.
В public/checkout.html заменяете: apikey=YOUR_YANDEX_MAPS_API_KEY на ваш ключ.
Настроить почты (Mail.ru SMTP)
Для каждой почты (y.underwear.order@mail.ru и y.underwear@mail.ru) в настройках Mail.ru включите SMTP/IMAP и создайте пароль приложения (если обычный пароль не работает).
Заполните в .env:
ORDER_EMAIL, ORDER_EMAIL_PASSWORD
CLIENT_EMAIL, CLIENT_EMAIL_PASSWORD


Тест локально
npm i nodemailer axios uuid
node server.js
Добавьте товары в админке.
Оформите заказ на /checkout.html
Система создаст заказ (pending) → отправит на оплату ЮKassa → после возврата на payment-success.html проверит статус и, если succeeded, отправит 2 письма.
